# Security policy

Byte Core is in pre-alpha repository bootstrap. No stable release exists, and the `main` branch is not production-supported.

Security reports about the current repository are welcome, but this policy does not promise support, response, or remediation timelines.

## Supported versions

| Version or branch | Status |
| --- | --- |
| `main` | Pre-alpha; security reports accepted, not production-supported |
| Stable releases | None published |

This table will be revised when Byte Core publishes supported releases.

## Report a vulnerability privately

Use [GitHub private vulnerability reporting](https://github.com/kodiakdirus/byte-core/security/advisories/new) to report a suspected vulnerability.

Do not disclose an unpatched vulnerability in a public issue, pull request, discussion, or other public channel.

GitHub private vulnerability reporting is the only private security-reporting route currently designated by this project. Byte Core does not publish a personal security email address.

## Information to include

Provide only the information needed to understand and reproduce the issue:

- a concise description of the suspected vulnerability;
- the affected revision, file, or component;
- minimal reproduction steps or a minimal proof of concept;
- the security impact you believe is possible; and
- a suggested mitigation, if you have one.

Use fictional or sanitized data whenever possible.

Do not include unrelated credentials, environment variables, private infrastructure inventory, real hostnames or addresses, broad logs, transcripts, or arbitrary file contents. If sensitive material is essential to the report, minimize it and explain why it is necessary.

## Coordinated disclosure

Please allow the issue to be investigated and, when appropriate, corrected before public disclosure.

The project may ask for clarification, determine that a report is not a security vulnerability, or coordinate later disclosure through a GitHub security advisory. No particular response, remediation, publication, or release timeline is guaranteed.

## Non-security reports

Report ordinary bugs and feature requests through the public [issue tracker](https://github.com/kodiakdirus/byte-core/issues).

Do not use private vulnerability reporting for conduct complaints. A separate private conduct-reporting process has not yet been published.
